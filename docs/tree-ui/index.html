TreeUI Manual

Version 1.1.0 // The service book for the TreeUI ecosystem. Optimized for performance, designed for safety.

TreeUI handles menu navigation using a high-performance tree structure. It extends Unity's native parent-child hierarchy to create a modular system with Dictionary-based O(n) traversal.

Built for the modern developer: Zero-GC runtime, Editor Undo/Redo support, and Structural Enforcement Guardrails.

The Network Root

Right-click in hierarchy: GameObject -> UI -> TreeUI -> Menu Network. This creates the Manager, Canvas, and Transitions root automatically.

Creating a Menu

Select a parent and use UI -> TreeUI -> Menu. A popup wizard handles naming and automated list registration.

Transitions are strictly Animator-based:

  • Add a StartTransition Trigger to your Animator.
  • Add an Animation Event calling Transition.OpenMenu on the desired frame.

Use OnMenuOpen and OnMenuClose UnityEvents to trigger modular logic per-menu.

MenuManager.cs

OpenMenu(string path, int sceneIndex) - Supports additive loading.

LoadLastMenu() - Reopens previous menu state.

Menu.cs

isOpen (bool) - Returns current visibility state.
