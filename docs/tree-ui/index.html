TreeUI Manual

The comprehensive service book for the TreeUI framework. Structured for rapid implementation and production-grade stability.

01. Overview

▼

TreeUI manages menu navigation using a high-performance tree architecture. It eliminates traditional search overhead by utilizing a Dictionary-based O(n) traversal system.

  • Zero Runtime GC Allocations
  • Integrated Scene Management (Additive Support)
  • Full Editor Undo/Redo Integration

02. Menu System Quick Start

▼

Creating the Network

Right-click in Hierarchy: GameObject -> UI -> TreeUI -> Menu Network. This automates the creation of the Manager, Canvas, and Transitions root.

Adding Menus

Select a parent and use UI -> TreeUI -> Menu. A setup wizard will manage naming and internal references.

03. Transitions

▼

Designer-friendly Animator workflow:

  1. Add a StartTransition Trigger to your Animator.
  2. Call Transition.OpenMenu via an Animation Event on your swap frame.

04. Interactions

▼

Hook into modular events via the Inspector:

  • OnMenuOpen: Triggered when the menu becomes active.
  • OnMenuClose: Triggered when the menu is disabled.

05. MenuManager - Script

▼

Core API

OpenMenu(string path, int sceneIndex): Navigation with additive scene support.

Supports LoadSceneMode.Additive for multi-scene UI architectures.

History API

LoadLastMenu(): Automatically traverses back one step in the navigation history stack.

06. Menu Script

▼

Attached to every UI Page. Handles visibility state and stores references to children/parents for the search algorithm.

public UnityEvent OnMenuOpen;

07. Transition - Script

▼

Acts as the bridge between Unity's Animator and MenuManager. Must be placed under the 'Transitions' root object.
