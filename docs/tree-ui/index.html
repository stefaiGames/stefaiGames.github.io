TreeUI Manual

The comprehensive service book for the TreeUI framework. Structured for rapid implementation and production-grade stability.

01. Overview

▼

TreeUI handles menu navigation using a high-performance tree structure. It extends Unity's native parent-child hierarchy to create a modular system with Dictionary-based O(n) traversal.

Features include Zero-GC runtime, integrated Scene Management, and self-healing hierarchy guardrails.

02. Menu System Quick Start

▼

Creating the Network

Right-click in Hierarchy: GameObject -> UI -> TreeUI -> Menu Network. This automates the creation of the Manager, Canvas, and Transitions root.

Adding Menus

Select a parent and use UI -> TreeUI -> Menu. A setup wizard will manage naming and automated registration.

03. Transitions

▼

Designer-friendly Animator workflow:

  1. Add a StartTransition Trigger to your Animator.
  2. Call Transition.OpenMenu via an Animation Event on your swap frame.

04. Interactions

▼

The system provides event hooks for modular logic. You can hook into OnMenuOpen and OnMenuClose via the Inspector to trigger sounds, analytics, or gameplay state changes.

05. MenuManager - Script

▼

Core API

OpenMenu(string path, int sceneIndex): The primary navigation method.

Update: Now supports additive scene loading for complex UI architectures.

History API

LoadLastMenu(): Automatically traverses back one step in the navigation history stack.

06. Menu Script

▼

Attached to every UI Page. Handles visibility state and stores references to children/parents for the search algorithm.

▼
