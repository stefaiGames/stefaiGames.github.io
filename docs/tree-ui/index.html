TreeUI Documentation

The following documentation covers the installation, architecture, and API reference for the TreeUI framework. Optimized for performance, designed for designers.

TreeUI is an optimized framework that tackles Unity's menu handling through a Tree-Based Structure. It extends Unityâ€™s native parent-child hierarchy to create a modular, scalable, and memory-efficient system.

  • Dictionary-based O(n) search logic
  • Zero/Minimal Runtime GC Allocations
  • Integrated Scene Management (Additive Ready)
  • Full Undo/Redo Friendly Editor Workflow
  • Animator-based Transition System
  • Self-Healing Hierarchy Guardrails

1. The Menu Network

Right-click in hierarchy or use GameObject -> UI -> TreeUI -> Menu Network. This creates the Manager, Canvas, and Transitions root. The system automatically registers these with Unity's Undo stack.

2. Creating Menus

Select your Manager or a Parent Menu and use UI -> TreeUI -> Menu. The Setup Wizard will guide you through naming. Note: Names must be unique within their branch.

Realist Pro-Tip: The "Fuzzy Search" will help debug typos if you call a menu path that doesn't exist.

Transitions are handled by Unity's Animator. This allows designers to tweak animations without touching code.

A

The Trigger

In your Animator Controller, create a Trigger parameter named StartTransition.

B

The Animation Event

On the frame where you want the menu to actually swap, add an Animation Event and call Transition.OpenMenu.

TreeUI includes per-menu events for modular logic. No more global scripts checking which menu is open.

  • OnMenuOpen: Triggered the moment the menu becomes active.
  • OnMenuClose: Triggered the moment the menu begins closing.
// Example: Accessing events via code
myMenu.OnMenuOpen.AddListener(() => PauseGame());

MenuManager.cs

  • OpenMenu(string path): Navigates to a menu using its hierarchical path.
  • LoadLastMenu(): Reopens the last active menu from history.
  • OpenMenu(string path, int sceneIndex): Supports additive scene loading.

Menu.cs

  • menuName: The unique identifier for this branch.
  • isOpen: Returns the current visibility state.
  • HandleRenaming(): Automatically manages dictionary keys on name changes.

Architecture

Create your menu network in a dedicated scene for DDOL objects to prevent singleton duplication on scene reloads.

Hierarchy

Keep names consistent and follow C# naming conventions. TreeUI enforces strict parenting logic, but a clean hierarchy starts with you.
