TreeUI Manual

Systematic Modernization of Unity UI Workflows. v1.1.0 (Production Ready)

TreeUI extends Unity's native parent-child hierarchy to create a high-performance tree structure. Unlike traditional systems that rely on recursive searches, TreeUI uses Dictionary-based traversal, achieving O(n) search time relative only to tree depth.

Technical Elite

  • 0 Runtime GC Allocation
  • 985 Lines of Lean Logic
  • Full Undo/Redo Support

Safety Built-in

  • Self-Healing Hierarchy
  • Fuzzy Search Error Handling
  • Structural Enforcement Logic

1. Create the Network

Right-click in the Hierarchy or go to GameObject -> UI -> TreeUI -> Menu Network. This automatically creates the MenuManager, Transitions root, and initial Canvas.

2. Launch the Setup Wizard

Select your parent (MenuManager or a Menu) and choose UI -> TreeUI -> Menu. A popup will appear allowing you to name the object and define its search path.

Realist Tip: The system will automatically undo any illegal parenting (placing a menu outside the network) and alert you in the console.

TreeUI uses the Standard Animator Workflow. Every transition is an Animator state, making it trivial to move keyframes or change timings.

The Mandatory Parameter

In your Animator Controller, you must add a Trigger named StartTransition (case sensitive). The system uses this to kick off your animation.

The Transition Event

On the specific frame where you want the old menu to disappear and the new one to appear, add an Animation Event and select Transition -> OpenMenu.

Additive Scene Loading

Transition between menus across scene boundaries. Support for LoadSceneMode.Additive ensures your UI persistent throughout the gameplay session.

Load Last Menu

Use MenuManager.Instance.LoadLastMenu() to automatically return the user to their previous state, including nested menu positions.

Contextual Events

Every menu contains OnMenuOpen and OnMenuClose UnityEvents. Hook in specific logic like pausing time, playing sounds, or updating player data directly in the Inspector.
